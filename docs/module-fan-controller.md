## 這個模組在做什麼？

伺服器機箱裡有很多風扇，需要根據溫度自動調整轉速：

- **溫度低**時，風扇轉慢一點 → 省電、安靜
- **溫度高**時，風扇全速運轉 → 保護硬體

這個模組就是做這件事的「控制板」。它接收 BMC（伺服器主機板管理控制器）的 I2C 指令，透過 EMC2103 晶片產生 PWM 訊號來控制風扇轉速。

```
BMC 主機板                    本模組                         風扇
┌──────────┐   I2C 匯流排   ┌──────────────┐   PWM 訊號   ┌──────┐
│  管理     │──────────────→│  EMC2103     │─────────────→│ 🌀   │
│  控制器   │←──────────────│  風扇控制器   │←─────────────│  轉速 │
└──────────┘  SDA/SCL      └──────────────┘  Tach 反饋   └──────┘
```

## 核心晶片：EMC2103

| 項目 | 說明 |
|------|------|
| **型號** | Microchip EMC2103 |
| **封裝** | QFN-20 (4mm × 4mm)，底部有散熱焊盤 |
| **供電** | 3.3V (VCC 和 VDDIO 都接 3.3V) |
| **通訊** | I2C / SMBus，地址 0x2E（ADDR_SEL 接地時） |
| **功能** | PWM 風扇控制 + 轉速監測 + 溫度感測 |
| **特殊** | SMBALERT# 開漏輸出，過溫時主動通知 BMC |

### 什麼是 QFN-20？

QFN = Quad Flat No-lead（四面無引腳扁平封裝）。「20」表示有 20 個接腳。這種封裝的腳位在晶片底部，不像傳統 IC 有外伸的腳，所以：
- ✅ 體積小、散熱好
- ❌ 需要 SMT（表面貼裝）製程，不能手焊

底部的大焊盤叫 **Exposed Pad (EP)**，要接到 GND，用來散熱。

## 電壓域詳解

> 在 Demo 的「📊 分析報告」→「🔌 電壓域 — Fan Controller」可以看到

這塊板有 **兩個電壓域**，用途完全不同：

### 3.3V 域（邏輯控制）
| Net | 說明 |
|-----|------|
| `3V3` | EMC2103 的供電（VCC + VDDIO）、I2C 上拉電阻的高電位 |
| `I2C_SDA` | I2C 資料線，透過 4.7kΩ 上拉到 3.3V |
| `I2C_SCL` | I2C 時脈線，透過 4.7kΩ 上拉到 3.3V |
| `FAN_PWM` | EMC2103 輸出的 PWM 訊號（0~3.3V 方波）|
| `FAN_TACH` | 風扇回傳的轉速脈衝（每轉 2 個脈衝）|
| `SMBALERT_N` | 過溫警報（低電位有效，平時被 10kΩ 拉到 3.3V）|

### 12V 域（風扇供電）
| Net | 說明 |
|-----|------|
| `12V` | 風扇馬達的供電，直接從 J1 Pin2 送到風扇 |

### GND（共用接地）
| Net | 說明 |
|-----|------|
| `GND` | 所有零件共用的 0V 參考電位，PCB 背面整面填銅 |

### ⚠️ 為什麼要分電壓域？

12V 是「力量大」的電源（驅動風扇馬達），3.3V 是「精細」的電源（IC 邏輯運算）。如果一個 3.3V 的腳位不小心接到 12V，IC 就會燒毀。這就是為什麼 Analyzer 要做「電壓守衛」檢查。

## 每個零件的功能說明

### U1: EMC2103 — 核心控制晶片

這是整塊板的「大腦」。它的腳位功能：

| 腳位 | 名稱 | 連接 | 功能 |
|------|------|------|------|
| 7 | VCC | 3V3 | 核心供電 |
| 8 | VDDIO | 3V3 | I/O 電壓準位 |
| 9 | GND | GND | 接地 |
| 21 | EP | GND | 散熱焊盤（底部） |
| 4 | SDA | I2C_SDA | I2C 資料線 |
| 5 | SCL | I2C_SCL | I2C 時脈線 |
| 6 | ADDR_SEL | GND | 地址選擇（接地 = 0x2E） |
| 3 | SMBALERT | SMBALERT_N | 過溫中斷通知 |
| 2 | FAN_PWM | FAN_PWM | PWM 輸出控制風扇 |
| 1 | FAN_TACH | FAN_TACH | 讀取風扇轉速 |

### R1, R2: I2C 上拉電阻 (4.7kΩ)

I2C 匯流排是**開漏 (Open-Drain)** 架構，意思是 IC 只能「拉低」線路，無法「拉高」。所以需要外部電阻把線路「拉高」到 3.3V，這就是上拉電阻的功能。

- **為什麼是 4.7kΩ？** 這是 I2C Standard Mode (100kHz) 的經典值，平衡了上升時間和功耗。
- **Demo 對照**：在原理圖的 🟢 綠色區塊 (B) 可以看到

### R3: SMBALERT# 上拉電阻 (10kΩ)

SMBALERT# 是「開漏」輸出（# 號表示低電位有效）。平時被 10kΩ 拉到 3.3V（高電位=沒事），當 EMC2103 偵測到溫度異常時，會把這條線拉到 0V，通知 BMC。

### C1, C2: 3.3V 去耦電容

- **C1 (100nF)** — 濾除高頻雜訊（MHz 等級），要盡可能靠近 U1 的 VCC 腳
- **C2 (10µF)** — 儲存能量，應對瞬間電流需求

**為什麼需要兩個？** 小電容反應快（濾高頻），大電容容量大（穩低頻）。這是 PCB 設計的基本功。

### C3, C4: 12V 去耦電容

同理，但用在 12V 風扇電源上：
- **C3 (100nF)** — 高頻濾波
- **C4 (100µF)** — 風扇啟動瞬間需要大電流，靠這顆電容補給

### J1: 4-Pin PWM 風扇接頭

標準 PC 風扇接頭的腳位定義：

| Pin | 功能 | 連接 |
|-----|------|------|
| 1 | GND | 接地 |
| 2 | 12V | 風扇供電 |
| 3 | Tach | 轉速回傳訊號 |
| 4 | PWM | 控制轉速的方波 |

### J2: BMC I2C 接頭

| Pin | 功能 |
|-----|------|
| 1 | SDA（I2C 資料） |
| 2 | SCL（I2C 時脈） |
| 3 | SMBALERT#（過溫通知） |
| 4 | GND |

## PCB 佈局設計要點

> 在 Demo 的「📐 PCB 2D 佈局」頁面可以查看，用方向鍵平移、滾輪縮放

| 項目 | 設計值 | 說明 |
|------|--------|------|
| 板尺寸 | 40mm × 30mm | 適合伺服器機箱子卡 |
| 層數 | 2 層 (F.Cu / B.Cu) | 正面走線，背面 GND 填銅 |
| 12V 走線寬度 | 0.6mm | 可承載 ~2A 風扇電流 |
| 3V3 走線寬度 | 0.4mm | 電源走線 |
| 訊號走線寬度 | 0.2mm | I2C / PWM / Tach |
| Via 孔徑 | 0.3mm (drill 0.2mm) | 訊號 Via |
| Thermal Via | 4 個在 U1 下方 | 導熱至背面 GND 銅皮 |
| 安裝孔 | 4 × M2.5 | 四角，離邊 3mm |

### GND 策略

背面 (B.Cu) 整層都是 GND 填銅。這有三個好處：
1. **降低雜訊** — 所有回流電流都有低阻抗路徑
2. **散熱** — U1 的 Exposed Pad 透過 Thermal Via 導熱到背面銅皮
3. **EMI 防護** — GND 平面像屏蔽層，減少電磁干擾

## 如何對照 Demo 學習

1. **先看原理圖** — 了解「電路怎麼連的」
   - 找到 U1，看它的每個腳位連到哪裡
   - 對照上方的腳位表，理解每個連接的目的

2. **再看 PCB 佈局** — 了解「零件放在哪裡」
   - 注意 U1 在中央，去耦電容緊貼它
   - 12V 走線比較粗，訊號線比較細

3. **看 3D 模型** — 了解「實物長什麼樣」
   - QFN-20 是黑色方塊，腳位在底部
   - Pin Header 是黑色直排插座
   - 0402 電阻/電容非常小（幾乎只看到焊盤）

4. **看分析報告** — 了解「為什麼這樣設計」
   - 電壓域表：確認每條線是什麼電壓
   - BOM 表：了解每個零件的成本
   - AI 建議：學習設計的思考邏輯
