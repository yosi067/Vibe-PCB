## 這個模組在做什麼？

在伺服器或 AI 訓練機中，GPU 會消耗大量的電力（一張卡可能吃 300~700W）。我們需要即時知道：
- 電源軌的 **電壓** 有沒有降低（代表供電不足）
- 流過的 **電流** 有多大（代表負載多重）
- 總 **功率** 是多少（防止超過預算或過熱）

這個模組就像電源的「體溫計」：

```
PSU 電源供應器           本模組                        GPU 負載
┌──────────┐          ┌──────────────┐             ┌──────────┐
│  12V DC  │──[J3]──→│ Rs (0.002Ω) │──[J4]────→ │  GPU     │
│          │          │      ↓        │             │  運算    │
│          │          │   INA226     │             │          │
│          │          │   量測 V/I/P  │             │          │
└──────────┘          └──────┬───────┘             └──────────┘
                             │ I2C
                             ↓
                          BMC 主機板
                        （讀取數據、發告警）
```

### 什麼是 MSOP-10？

MSOP = Mini Small Outline Package（迷你小型輪廓封裝）。跟 SOP（SOIC）很像但更小。10 個腳位分兩排，每排 5 個，腳距 0.5mm。

- ✅ 體積比 QFN 稍大，但腳伸出來可以目視檢查焊接
- ✅ 標準 SMT 製程

## 電流量測原理

INA226 使用 **分流電阻法** 來量測電流。這是最基礎也最精確的方法：

### 歐姆定律

$$V = I \times R$$

如果我們放一個已知阻值的電阻 $R_s$ 在電流路徑上，量測它兩端的電壓差 $V_{shunt}$，就能算出電流：

$$I = \frac{V_{shunt}}{R_s}$$

### 在本模組中

- **分流電阻** $R_s$ = 0.002Ω (2mΩ)
- **最大電流** 30A 時：$V_{shunt}$ = 30 × 0.002 = **60mV**
- INA226 可以量測 ±81.92mV 的分流電壓，所以 60mV 在範圍內 ✔

### 為什麼用這麼小的電阻？

電阻越大越好量（電壓差大），但電阻上會「浪費」能量。在大電流場景中：

$$P = I^2 \times R$$

如果 $R_s$ = 0.01Ω（10mΩ），30A 時功耗 = $30^2 \times 0.01$ = **9W** 😱  
如果 $R_s$ = 0.002Ω（2mΩ），30A 時功耗 = $30^2 \times 0.002$ = **1.8W** ✔

所以要在「好量測」和「低功耗」之間取平衡。0.002Ω 是常見選擇。

## 電壓域詳解

> 在 Demo 的「📊 分析報告」→「🔌 電壓域 — Power Monitor」可以看到

這塊板有 **三個電壓域**：

### 3.3V 域（邏輯控制）
| Net | 說明 |
|-----|------|
| `3V3` | INA226 的供電 (VS)、I2C 上拉、Alert 上拉 |
| `I2C_SDA` | I2C 資料線 |
| `I2C_SCL` | I2C 時脈線 |
| `PWR_ALERT_N` | 告警輸出，低電位有效 |

### 12V 域（被監測的功率路徑）
| Net | 說明 |
|-----|------|
| `12V_INPUT` | 從 PSU 進來的 12V，經過 Rs1 的 **高電位端** |
| `12V_OUTPUT` | 經過 Rs1 後送往 GPU 的 12V，Rs1 的 **低電位端** |

> 注意：`12V_INPUT` 和 `12V_OUTPUT` 的電壓差只有 **60mV**（30A 時），幾乎一樣。分成兩個 Net 是為了讓 INA226 量測這個微小的差值。

### GND（共用接地）
| Net | 說明 |
|-----|------|
| `GND` | 所有零件共用的 0V 參考 |

## 每個零件的功能說明

### U2: INA226 — 高精度電流感測 IC

| 腳位 | 名稱 | 連接 | 功能 |
|------|------|------|------|
| 6 | VS | 3V3 | IC 供電 |
| 7 | GND | GND | 接地 |
| 10 | IN+ | 12V_INPUT | 分流電阻高電位端（感測輸入+）|
| 9 | IN- | 12V_OUTPUT | 分流電阻低電位端（感測輸入-）|
| 8 | VBUS | 12V_OUTPUT | 匯流排電壓量測（量測負載端電壓）|
| 4 | SDA | I2C_SDA | I2C 資料 |
| 5 | SCL | I2C_SCL | I2C 時脈 |
| 3 | ALERT | PWR_ALERT_N | 過流/過壓告警 |
| 2 | A0 | 3V3 | 地址位元 0 = 1 |
| 1 | A1 | GND | 地址位元 1 = 0 |

### I2C 地址怎麼決定的？

INA226 的 I2C 地址由 A0 和 A1 兩個腳位決定：

| A1 | A0 | 地址 |
|----|-----|------|
| GND | GND | 0x40 |
| GND | VCC | **0x41** ← 我們的設定 |
| VCC | GND | 0x44 |
| VCC | VCC | 0x45 |

我們用 **0x41**，是因為如果同一條 I2C 匯流排上還有風扇控制器（0x2E），兩者不會衝突。Analyzer 裡的「I2C 位址衝突偵測」就是在檢查這件事。

### Rs1: 精密分流電阻 (0.002Ω, 2512)

這是整個模組最關鍵的零件！

| 項目 | 值 | 說明 |
|------|-----|------|
| 阻值 | 0.002Ω (2mΩ) | 極低阻值，減少壓降和功耗 |
| 封裝 | 2512 (6.3mm × 3.2mm) | 大封裝 = 大功率承受能力 |
| 額定功率 | 2W | 封裝的最大散熱能力 |
| 實際功耗 | 1.8W (@ 30A) | 有 0.2W 餘量 |
| 精度 | ±1% 或更好 | 量測精度取決於電阻精度 |

### D1 + R7: Alert LED 指示

ALERT 腳是開漏輸出，正常時被 R6 (10kΩ) 拉高到 3.3V。LED 的電路：

```
3V3 ──[R6 10k]──┬── ALERT (U2 pin3)
                 │
                 ├──[R7 1k]──[D1 LED]──→ GND
                 │
              PWR_ALERT_N net
```

- **正常狀態**：ALERT = 3.3V，LED 兩端壓差不夠，不亮
- **告警時**：INA226 將 ALERT 拉到 GND，電流經 R7 和 LED 流到 GND，LED 亮 🔴

LED 電流計算：$(3.3V - 2V_{LED}) / 1kΩ ≈ 1.3mA$（微亮，省電）

## 分流電阻的設計計算

> 💡 這是 PCB 設計中最重要的「選型計算」之一，Demo 的分析報告中也有 P=I²R 檢查

### 步驟 1：決定最大電流

我們的設計目標是監測 **最大 30A** 的 12V 電源。

### 步驟 2：選擇分流電阻值

INA226 的分流電壓量測範圍是 ±81.92mV。為了留餘量，我們希望滿載時分流電壓不超過 80mV。

$$R_s = \frac{V_{shunt,max}}{I_{max}} = \frac{80mV}{30A} ≈ 0.00267Ω$$

取標準值 **0.002Ω** (偏保守，$V_{shunt}$ = 60mV)。

### 步驟 3：計算功耗

$$P_{shunt} = I_{max}^2 \times R_s = 30^2 \times 0.002 = 1.8W$$

### 步驟 4：選擇封裝

| 封裝 | 額定功率 | 夠不夠？ |
|------|----------|---------|
| 0402 | 0.0625W | ❌ 完全不夠 |
| 0603 | 0.1W | ❌ 不夠 |
| 0805 | 0.125W | ❌ 不夠 |
| 1206 | 0.25W | ❌ 不夠 |
| 2010 | 0.75W | ❌ 不夠 |
| **2512** | **2W** | **✅ 1.8W < 2W，有 0.2W 餘量** |

⚠️ **注意**：0.2W 的餘量其實偏少（只有 10%）。工程上建議至少 50% 餘量。如果預算允許，可以用 2 個 0.004Ω 並聯（等效 0.002Ω，但功率額定 4W）。

### 🔑 大電流走線設計

30A 的電流在 PCB 上需要非常寬的走線！根據 IPC-2221 標準（1oz 銅厚，10°C 溫升）：

$$W = \frac{I}{k \times \Delta T^b \times t^c}$$

簡化經驗法則：1oz 銅厚下，每安培需要約 **0.5mm** 走線寬度（外層），所以 30A 需要至少 **15mm** 寬度——這在 35mm 寬的板上幾乎不可能！

**實際做法**：
- 12V 走線做 1.5mm 寬（正面）
- 背面也有對應的 GND 回流銅皮
- 實際量產時會加厚銅箔（2oz 或 3oz）
- 或者使用匯流排條 (busbar) 替代 PCB 走線

### 🔑 Kelvin 連接

分流電阻的感測線 (IN+/IN-) 必須使用 **Kelvin 連接**（四線量測法）：

```
        大電流走線 (粗)
  J3 ═══════[Rs1]═══════ J4
            │    │
            │    │  感測線 (細)
            │    └──→ U2 IN-
            └──────→ U2 IN+
```

感測線直接從電阻的焊盤引出，不要從大電流走線上分支。這樣量測的是電阻本身的壓降，而不是走線的壓降（走線也有電阻！）。

## 如何對照 Demo 學習

### 學習路線建議

#### 第一步：理解電流量測原理
1. 閱讀本文的 [電流量測原理](#電流量測原理) 章節
2. 在 Demo 原理圖中找到 **Rs1**（🔴 紅色區塊 A）
3. 注意 Rs1 兩端分別連接 `12V_INPUT` 和 `12V_OUTPUT`

#### 第二步：追蹤感測路徑
1. 在原理圖中找到 U2 的 **IN+** 和 **IN-** 腳位
2. 確認它們分別連到 Rs1 的兩端
3. **VBUS** 腳位接到 12V_OUTPUT — 這是用來量「匯流排電壓」的

#### 第三步：理解 I2C 位址
1. 找到 U2 的 A0 和 A1 腳位
2. A0 → VCC (=1)，A1 → GND (=0)
3. 查表：地址 = **0x41**
4. 對照分析報告的「I2C 位址衝突偵測」

#### 第四步：理解告警機制
1. 找到 U2 的 ALERT 腳位
2. 追蹤：ALERT → R6 上拉到 3.3V → R7 + D1 到 GND
3. 想一想：正常時 LED 亮還是不亮？

#### 第五步：比較兩個模組
| 比較項目 | Fan Controller | Power Monitor |
|----------|---------------|---------------|
| 核心 IC | EMC2103 (QFN-20) | INA226 (MSOP-10) |
| I2C 地址 | 0x2E | 0x41 |
| 大電流零件 | 無（12V 只供風扇） | Rs1 (30A!) |
| 板尺寸 | 40×30mm | 35×25mm |
| 特殊設計 | Thermal Via 散熱 | Kelvin 連接 |
| 複雜度 | 入門 | 中等 |

### 常見初學者問題

**Q: 為什麼 I2C 上拉電阻在兩個模組都有？不會衝突嗎？**
A: 在實際應用中，同一條 I2C 匯流排上只需要一組上拉電阻。這裡每個模組獨立設計，如果兩個模組共用匯流排，需要只保留一組上拉。

**Q: 0.002Ω 的電阻怎麼焊？看起來跟短路一樣**
A: 確實，分流電阻的阻值很低，外觀跟跳線差不多。但它內部使用特殊合金材料，有精確的阻值和很低的溫度係數。

**Q: 為什麼 INA226 供電用 3.3V 但可以量 12V？**
A: INA226 內部有分壓和保護電路。VBUS 腳位雖然接 12V，但 IC 內部會除以一定比例。IN+/IN- 腳位量測的是只有幾十 mV 的差分電壓，不會超過 IC 的承受範圍。

**Q: Analyzer 裡的 P=I²R 檢查是什麼？**
A: 它會自動計算每個電阻在最大電流下的功耗，檢查是否超過封裝的額定功率。例如 Rs1 的 1.8W < 2W (2512) 就會 PASS，但如果有人不小心用 0402 封裝（只有 0.0625W），就會被警告。
